# NEURAL TOKYO — 開発ログ（再現記事の素材・2026-07-05）

## 発想プロセス
1. **リサーチ**: 2025-26のバズ構造を分析（Lütke CT viewer 600万view / Dogan「1年を1時間」/ MediaPipe×Three.js / Genie 3）。共通項＝ (a)3秒で「生きている」 (b)常識破壊キャプション (c)ブラウザで即体験 (d)単一ファイル再現性。
2. **発散**: 20案（ログ: 会話履歴のStep 4）→ 10軸評価 → 上位3案を視覚比較 → Hirakuの「技術的誠実性を担保できるか？」の問いで再評価 → 新4案追加提示。
3. **決定**: 案D SWARM CITY（交通科学・誠実性満点）× 案A NEURAL DRIVE誠実版（ピクセル入力CNN模倣学習）の融合。「都市の群れ」と「学ぶ1台」の対比構造。

## 技術選定
- Three.js r160のみ（importmap/CDN）。NN・交通・都市生成・雨・ポストFXは全て自作 → 「no ML library」という投稿上の一撃と、単一ファイル配布性のため。
- NN: 769→48→24→2 MLP（38,186パラメータ）、純JSバックプロパゲーション＋SGDモメンタム。入力は本物のWebGL POVレンダリング（96×32→48×16グレースケール）。
- 交通: IDM（Treiber 2000）＋時分割式無信号交差点調停＋人間/AVパラメータプロファイル。
- 学習: behavioral cloning＋逸脱時教師介入（DAgger的）。介入回数・無介入走行距離は実測値。

## 実装で苦労した点（記事のネタ）
1. **左側通行の座標系バグ**: up×forward=left の計算で、Z軸道路とX軸道路で符号が逆になる（z: +dir, x: -dir）。スクショで2回間違いを検出し、外積を書き下して確定。
2. **交差点のグリッドロック**: 「他軸がアイドルになったら切替」ロジックは高密度で永久ブロックする。maxHold付き時分割に変更 → AV率で保持時間を補間（av²で2.2s→7s）。この1変更で 100%AV=27km/h vs 100%人間=13.8km/h の劇的コントラストが生まれた。
3. **夜景のフォトリアル**: ベイクした道路テクスチャは4.5px/mで破綻 → 車線・横断歩道をジオメトリ化。窓は「セル+枠+床スラブ+カーテン+1F店舗」のcanvas生成。濡れ路面は環境マップでは前方光源を拾えない → ライトストリーク（加算スプライト）方式へ。
4. **俯瞰で交通が見えない**: ライトは垂直クアッドで真上から不可視 → 白→赤グラデの「トップグロー」スプライトを追加し、70m以上のカメラ高度でのみ表示。
5. **QAループ**: サンドボックスにChromiumをplaywright経由で導入（libXdamage欠損はスタブ.soを自作コンパイルして解決）、CDNは403のためローカルthree.jsへルートインターセプト。`?t=60`（決定論的ウォームアップ）と`?shot=`（固定カメラ）をQA用に本体へ実装 — これがそのまま撮影機能になった。
